Native Federation Demo

Shell Application — Angular 20

How it works

This shell application is the host. It handles authentication, navigation, and orchestration. Sub-applications are loaded at runtime via loadRemoteModule using Native Federation.

Independent deployment

Each sub-app is built and deployed separately. The shell reads federation.manifest.json at startup to discover remote entry points. No rebuild of the shell is needed when a sub-app is redeployed.

Shared singleton services

The AuthService from @skv/shared is a singleton across the entire federation. Log in here in the shell and navigate to the sub-app — it will see the same authenticated user.
