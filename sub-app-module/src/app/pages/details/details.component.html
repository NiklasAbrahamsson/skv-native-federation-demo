Remote micro frontend — child route (NgModule)

This page demonstrates that the NgModule-based sub-app manages its own child routes. The URL is /sub-app-module/details/{{ id }} — the shell's router delegates everything under /sub-app-module to this remote's RemoteModule which uses RouterModule.forChild().

The route parameter :id is read via ActivatedRoute.snapshot.paramMap — the traditional NgModule approach. Compare with the standalone sub-app which uses input.required() with withComponentInputBinding().
